# Head of Data 102: Running Coach

A Streamlit app where a runner talks to a coach in plain language, and the coach turns that conversation into a personalised training plan.

## What it does

The app has four screens:

- **Chat**: the runner describes their goal in conversation, the coach asks clarifying questions
- **Current plan**: the structured training plan generated by the agent
- **User profile**: the runner's basic info, goals, and constraints
- **Training history**: the runner's past sessions, loaded from the database

The chat pipeline extracts a structured goal from what the runner says. A LangGraph agent reads the runner's training history from the database via a tool call, then generates a validated week-by-week plan as structured JSON. If the runner mentions an injury or a race date change mid-conversation, a reschedule router detects it and triggers the agent automatically without requiring the button.

## Database schema

The app uses a remote Postgres database hosted on Supabase. Four tables:

```
users            (id, first_name, city, gender, goals, constraints, created_at)
training_history (id, user_id, session_date, km, pace, session_type, created_at)
plans            (id, user_id, plan_text, goal_json, created_at)
messages         (id, user_id, role, content, created_at)
```

Full create table statements are in `data/schema.sql`.

## Quick start

<details>
<summary><b>Setup and run, click to expand</b></summary>

### Prerequisites
- Python 3.10+
- A Gemini API key
- Access to the team Supabase project

### Setup

1. **Clone the repo**
```bash
git clone https://github.com/stuckingravity/hod102-2026-group-4.git
cd hod102-2026-group-4
```

2. **Create virtual environment**
```bash
python -m venv venv
source venv/bin/activate   # macOS/Linux
venv\Scripts\activate      # Windows
```

3. **Install dependencies**
```bash
pip install -r requirements.txt
```

4. **Create a `.env` file** in the project root:
```
GEMINI_API_KEY=your_gemini_key_here
APP_PASSWORD=pick_any_password_you_want
DATABASE_URL=postgresql://postgres:your_password@your_pooler_host:6543/postgres
```
Use the transaction pooler connection string from Supabase, not the direct hostname.

5. **Run the app**
```bash
streamlit run app/Home.py
```

6. **Run tests**
```bash
python tests/test_db.py
python tests/test_agent.py
```

## Open questions

- Chat history is stored in the messages table per turn but not loaded back into context on session start. Doing so safely requires a windowing or retrieval strategy to avoid exceeding the model context window.
- Editing the profile or training history is not built. The Modify button on the Profile page is a disabled placeholder.
- The conversational turns (send_message, extract_goal) and the agent are currently two separate pipelines. A full migration would route every user message through the LangGraph graph, with send_message as a third END path. This requires streaming intermediate agent messages back to the Streamlit UI during graph execution.

## Tech stack

**Core:** Python 3.10, Streamlit 1.58

**LLM:** google-genai 2.8.0, model gemini-3.1-flash-lite (15 RPM, 500 RPD, 250K TPM on free tier)

**Agent:** LangGraph 1.2.6, Pydantic 2.x for structured output validation

**Database:** Supabase (managed Postgres), psycopg2-binary 2.9.12 via transaction pooler

**Config:** python-dotenv
